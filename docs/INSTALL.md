# 安装指南

## 📋 系统要求

### 支持的浏览器
- ✅ **Google Chrome** 88.0.4324.104 或更高版本
- ✅ **Microsoft Edge** 88.0.705.81 或更高版本
- ✅ **其他基于Chromium的浏览器**

### 操作系统支持
- ✅ **Windows** 10/11
- ✅ **macOS** 10.14 或更高版本
- ✅ **Linux** (Ubuntu, CentOS, Debian等)

## 📦 安装方法

### 方法一：开发者模式安装（推荐）

1. **下载项目文件**
   - 下载或克隆项目到本地
   - 确保包含 `manifest.json` 文件

2. **启用开发者模式**
   - 打开Chrome浏览器
   - 访问 `chrome://extensions/`
   - 开启右上角的"开发者模式"开关

3. **加载扩展**
   - 点击"加载已解压的扩展程序"
   - 选择项目文件夹
   - 点击"选择文件夹"

4. **验证安装**
   - 扩展列表中应出现"Google Authenticator Extension"
   - 浏览器工具栏显示扩展图标
   - 点击图标能正常打开弹出窗口

### 方法二：使用打包脚本

```bash
# Linux/macOS
./scripts/package.sh

# Windows
scripts/package.bat
```

## 🚀 使用指南

### 首次使用
1. **点击扩展图标**：在浏览器工具栏点击扩展图标
2. **添加第一个账户**：
   - 点击"+ 添加账户"按钮
   - 输入密钥和账户名称
   - 或使用扫描功能添加账户

### 主要功能
- **查看验证码**：扩展自动显示当前有效的6位验证码
- **复制验证码**：点击验证码可直接复制到剪贴板
- **筛选账户**：使用搜索框按域名或账户名筛选
- **管理账户**：点击"⋯"菜单可以删除账户

### 添加账户的四种方式
1. **手动输入**：直接输入密钥和账户名称
2. **扫描二维码**：扫描网页上的二维码
3. **导入图片**：上传包含二维码的图片文件
4. **拖拽图片**：直接拖拽图片到指定区域

## ❓ 常见问题

### Q1: 扩展无法加载怎么办？
**A:** 请检查以下几点：
- 确保所有必需文件都存在
- 检查manifest.json文件格式是否正确
- 确保Chrome版本支持Manifest V3
- 尝试重新加载扩展

### Q2: 二维码扫描功能不工作？
**A:** 可能的原因和解决方案：
- **权限问题**：确保已授予摄像头权限
- **HTTPS要求**：某些功能需要HTTPS环境
- **浏览器兼容性**：确保使用支持的浏览器版本

### Q3: 验证码显示错误？
**A:** 检查以下项目：
- 确保系统时间正确
- 验证密钥格式是否正确
- 尝试重新添加账户

### Q4: 扩展图标不显示？
**A:** 解决方案：
- 检查扩展是否已启用
- 尝试固定扩展到工具栏
- 重新加载扩展

### Q5: 数据丢失怎么办？
**A:** 数据安全：
- 所有数据存储在本地，不会上传到服务器
- 建议定期备份重要账户信息
- 可以导出账户数据（如果支持）

## 🗑️ 卸载方法

### 完全卸载
1. 打开Chrome扩展管理页面（`chrome://extensions/`）
2. 找到"Google Authenticator Extension"
3. 点击"移除"按钮
4. 确认删除操作

### 数据清理
- 扩展卸载后，本地存储的数据会被自动清理
- 如需保留数据，请先导出账户信息

## 🔄 更新方法

### 开发者模式更新
1. 下载新版本文件
2. 在扩展管理页面点击"重新加载"按钮
3. 或删除旧版本后重新安装

### 自动更新
- 如果通过Chrome Web Store安装，会自动更新
- 开发者模式安装需要手动更新

## 🔒 安全说明

### 数据安全
- ✅ 所有数据仅存储在本地浏览器中
- ✅ 不会向任何外部服务器发送数据
- ✅ 使用标准的TOTP算法，确保安全性

### 权限说明
- **storage**：用于保存账户信息
- **activeTab**：用于扫描当前页面的二维码
- **scripting**：用于注入内容脚本
- **camera**：用于摄像头扫描二维码
- **alarms**：用于定时更新验证码

### 隐私保护
- 扩展不会收集任何个人信息
- 不会跟踪用户行为
- 所有计算都在本地进行
