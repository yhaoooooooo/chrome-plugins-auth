## 🔧 系统要求

### 支持的浏览器
- ✅ **Google Chrome** 88.0.4324.104 或更高版本
- ✅ **Microsoft Edge** 88.0.705.81 或更高版本
- ✅ **其他基于Chromium的浏览器**

### 操作系统支持
- ✅ **Windows** 10/11
- ✅ **macOS** 10.14 或更高版本
- ✅ **Linux** (Ubuntu, CentOS, Debian等)

- **摄像头**：用于二维码扫描功能（可选）

- **存储空间**：至少 50MB 可用空间

## 📦 安装方法

### 方法一：开发者模式安装（推荐）

这是最简单的安装方法，适合开发者和高级用户。

#### 步骤 1：下载项目文件
1. 下载或克隆项目到本地：
   ```bash
   git clone <项目地址>
   # 或者直接下载ZIP文件并解压
   ```

2. 确保项目目录包含以下文件：
   ```
   chrome-tools/
   ├── manifest.json
   ├── popup.html
   ├── popup.js
   ├── background.js
   ├── content.js
   ├── js/
   │   ├── authenticator.js
   │   ├── jsQR.js
   │   └── qrcode.min.js
   └── icons/
       ├── icon16.png
       ├── icon48.png
       └── icon128.png
   ```

#### 步骤 2：启用开发者模式
1. 打开Chrome浏览器
2. 在地址栏输入：`chrome://extensions/`
3. 在页面右上角找到"开发者模式"开关
4. 点击开关启用开发者模式

#### 步骤 3：加载扩展
1. 点击"加载已解压的扩展程序"按钮
2. 选择项目文件夹（包含manifest.json的目录）
3. 点击"选择文件夹"

#### 步骤 4：验证安装
1. 扩展列表中应该出现"Google Authenticator Extension"
2. 浏览器工具栏应该显示扩展图标
3. 点击图标应该能正常打开弹出窗口

### 方法二：打包安装

如果您希望分发扩展或进行更正式的安装，可以打包成.crx文件。

#### 步骤 1：打包扩展
1. 在Chrome扩展管理页面（`chrome://extensions/`）
2. 找到已安装的"Google Authenticator Extension"
3. 点击"打包扩展程序"按钮
4. 选择项目根目录作为扩展根目录
5. 点击"打包扩展程序"

#### 步骤 2：安装打包文件
1. 打包完成后会生成.crx文件
2. 将.crx文件拖拽到Chrome扩展管理页面
3. 确认安装对话框，点击"添加扩展程序"

## 🚀 使用指南

### 首次使用
1. **点击扩展图标**：在浏览器工具栏点击扩展图标
2. **添加第一个账户**：
   - 点击"+ 添加账户"按钮
   - 在新页面中输入密钥和账户名称
   - 或使用扫描功能添加账户

### 主要功能
- **查看验证码**：扩展会自动显示当前有效的6位验证码
- **复制验证码**：点击验证码可直接复制到剪贴板
- **筛选账户**：使用搜索框按域名或账户名筛选
- **管理账户**：点击"⋯"菜单可以删除账户

### 添加账户的四种方式
1. **手动输入**：直接输入密钥和账户名称
2. **扫描二维码**：扫描网页上的二维码
3. **导入图片**：上传包含二维码的图片文件
4. **拖拽图片**：直接拖拽图片到指定区域

## ❓ 常见问题

### Q1: 扩展无法加载怎么办？
**A:** 请检查以下几点：
- 确保所有必需文件都存在
- 检查manifest.json文件格式是否正确
- 确保Chrome版本支持Manifest V3
- 尝试重新加载扩展

### Q2: 二维码扫描功能不工作？
**A:** 可能的原因和解决方案：
- **权限问题**：确保已授予摄像头权限
- **HTTPS要求**：某些功能需要HTTPS环境
- **浏览器兼容性**：确保使用支持的浏览器版本

### Q3: 验证码显示错误？
**A:** 检查以下项目：
- 确保系统时间正确
- 验证密钥格式是否正确
- 尝试重新添加账户

### Q4: 扩展图标不显示？
**A:** 解决方案：
- 检查扩展是否已启用
- 尝试固定扩展到工具栏
- 重新加载扩展

### Q5: 数据丢失怎么办？
**A:** 数据安全：
- 所有数据存储在本地，不会上传到服务器
- 建议定期备份重要账户信息
- 可以导出账户数据（如果支持）

## 🗑️ 卸载方法

### 完全卸载
1. 打开Chrome扩展管理页面（`chrome://extensions/`）
2. 找到"Google Authenticator Extension"
3. 点击"移除"按钮
4. 确认删除操作

### 数据清理
- 扩展卸载后，本地存储的数据会被自动清理
- 如需保留数据，请先导出账户信息

## 🔒 安全说明

### 数据安全
- ✅ 所有数据仅存储在本地浏览器中
- ✅ 不会向任何外部服务器发送数据
- ✅ 使用标准的TOTP算法，确保安全性

### 权限说明
- **storage**：用于保存账户信息
- **activeTab**：用于扫描当前页面的二维码
- **scripting**：用于注入内容脚本
- **camera**：用于摄像头扫描二维码
- **alarms**：用于定时更新验证码

### 隐私保护
- 扩展不会收集任何个人信息
- 不会跟踪用户行为
- 所有计算都在本地进行
